## 요약

플레이테스트 피드백을 반영해 게임 언어를 "단어 전용"으로 재정립. 규칙 안내 화면, 단어 기반 파트너 응답, 세션마다 쌓이는 ASCII 아트 배경을 도입. 기본 모델을 flash-lite로 전환하고, 재시작 시 화면 잔여물이 남던 버그도 함께 수정.

## 결정 사항

- **단어로만 소통을 게임 언어로 격상**: 이전 버전은 파트너가 짧은 문장("오, 좋은데요?")을 뱉어 플레이어의 단어 입력과 비대칭했음. 이제 양측 모두 단어 배열로 말풍선 형태가 같아 시각적으로 '같은 규칙을 따르는 두 사람'처럼 보임
- **narrative는 채팅에서 제거, WORLD 패널 전용**: 채팅에 긴 묘사가 섞이면 단어 리듬이 깨짐. 묘사는 세계의 '결과물'로서 WORLD에만. 채팅은 인터랙션, WORLD는 세계. 역할 분리가 명확해짐
- **기본을 lite로 바꾸면서 프롬프트를 한국어로 재작성**: flash보다 instruction following이 약한 lite에 긴 영어 프롬프트를 주면 지시를 흘림. 한국어로 짧고 명시적으로 다시 쓰고, good/bad 예시를 모든 제약에 붙여 few-shot 유도
- **ASCII 아트를 "누적" 방식으로**: 교체가 아니라 겹치기. 초반엔 드문드문, 4세션 후에는 네 층이 겹쳐 빽빽해지는 시각적 성장 = 세계가 레이어로 쌓인다는 개념의 직관적 표현. 최신이 가장 진하고 이전 것이 그림자로 남는 opacity 차등은 플레이어가 '어디까지 만들었는지'를 스스로 느끼게 함
- **ASCII 레이어 color는 생성 당시 mood 고정**: 매번 최신 mood로 덮으면 네 층이 다 같은 색이 되어 누적 효과가 안 보임. 각 레이어가 그 시점 분위기를 간직하고, 전체는 겹쳐진 색 팔레트가 됨
- **패널 반투명: 채팅 0.85, WORLD 0.60**: 채팅은 글자 가독성이 최우선이라 더 불투명. WORLD는 세계 묘사라 배경이 더 비쳐 세계와 묘사가 한 화면에서 녹아 보이도록. 스펙 그대로
- **`new-world-btn`에서 `rules-screen` 스킵**: 규칙은 처음 오는 사람을 위한 onboarding. 새 세계 만들기를 누를 때는 이미 규칙을 아는 사용자. 재노출은 귀찮음만 늘림
- **폰트 크기 자동 계산에서 width/height 중 작은 값 채택**: 가로 기준 계산값이 세로 넘치면 잘리므로 안전한 쪽을 택함. 모바일 세로 화면에서도 30×15가 화면을 벗어나지 않음

## 미해결 / 다음 할 일

- **ASCII 30×15 스키마를 lite가 얼마나 지킬지 미검증**: 프롬프트에 "정확히 30자 × 15행"을 명시하고 이전 ASCII를 예시로 제공했지만, 행 길이가 들쭉날쭉할 가능성. 실플레이에서 깨지면 클라이언트 측 normalize(패딩/크롭) 추가 필요
- **ASCII에 줄바꿈 특수문자가 정확히 `\n`으로 올지**: JSON string에서 `\n`은 OK지만 모델이 `\\n`(백슬래시+n)로 쓸 수도 있음. 실측 후 재파싱 로직 고려
- **단어 기반 대화가 "대화처럼 느껴지는지" 실플레이 감각 체크**: 단어 배열만 오고 가면 오히려 기계적으로 읽힐 위험. 말풍선 타이핑 효과가 이를 얼마나 완화하는지, 타자 속도가 단어 길이에 비해 빠르게 느껴지는지 관찰
- **ASCII 전환 중 성능**: 4세션 누적 + 각 레이어에 opacity 2s 트랜지션 동시 작동. 모바일 저성능 기기에서 프레임 드롭 확인 필요
- **반전/공유 화면에 ASCII 배경이 남는 효과**: 의도된 연속성이지만, 반전 메시지("저는 사람이 아니에요")와 배경 ASCII가 시각적으로 경쟁할 수 있음. 필요하면 twist-screen 진입 시 컨테이너 opacity 감소 옵션 검토

## 세션 2 — 온보딩 호흡 보정 (프로필 로딩 / 인사는 문장 / 세션 경계는 시스템 알림)

### 맥락
첫 세션 변경 직후 플레이 감각 피드백 3가지:
1. 소개 입력 완료 → 규칙 화면이 "딸깍" 바로 뜸. 플레이어 입력이 어디로 갔는지 호흡이 없음
2. 파트너 첫 인사까지 단어 배열로 나오니, 게임 규칙이 적용되기도 전부터 이미 "단어만 쓰는 사이"처럼 읽힘 — 상대가 기계처럼 느껴짐
3. 세션 종료 시 "계속할까요?"가 파트너 말풍선으로 나와서, 상대가 게임 진행자를 겸하는 어색한 포지션이 됨

### 주요 변경 (`index.html`)

**1) 프로필 전송 로딩 화면 (`sending-screen`)**
- intro-screen 완료 → `playSendingProfile()` (1.6초 progress bar + 0.5초 여백) → rules-screen
- 매칭 화면과 동일한 바/커서 UI 재사용, 문구만 `"프로필을 보내는 중"`으로 차별화
- 실제 네트워크 호출은 없음 (의도적 가짜 딜레이). AI 정체 은폐 원칙에 맞춰 "서버·API" 같은 기술 용어는 문구에서 배제

**2) 첫 인사 문장 복원 (`addPartnerSentence`)**
- `greeting_words` → `greeting` 문자열로 롤백. `FALLBACK_GREETING_WORDS` → `FALLBACK_GREETING = '잠깐 이야기하다가 세계 만들어볼까요'`
- state: `firstGreetingWords` 제거, `firstGreeting` 문자열 복원
- 프롬프트에 "세계 만들기가 시작되면 단어로만 소통하지만, 지금은 시작 전"이라고 명시해 문장 인사만 나오도록 유도
- 렌더링: partner-screen은 원래부터 typewriter로 단순 문자열을 찍어주니 그대로. 세션 첫 진입의 채팅 첫 메시지는 `addPartnerSentence` (테두리 없음, 본문 폰트)로 바꿔, 이후 등장하는 단어 말풍선(`.chat-msg.partner` 기본: 테두리 + Mono)과 시각적으로 구분
- 단어 규칙 적용 시점: **세계 만들기가 시작된 순간부터**. 그 이전의 인사만 문장

**3) 세션 종료 경계를 시스템 알림으로 (`addSystemMsg`)**
- 파트너 말풍선 `addPartnerWords(['계속', '할까요'], ...)` 제거
- `.chat-msg.system` CSS 신규: 중앙 정렬, 흐린 색, Mono, letter-spacing 0.15em, 상하 얇은 선(`border-top`/`border-bottom`)
- 문구: `'계속할까요?'` (중간 세션) / `'다 만들었어요'` (마지막 세션)
- 채팅 흐름이 "파트너 대화 → 시스템 알림 → 버튼" 순으로 계층이 분리됨. 파트너는 더 이상 진행자 역할을 겸하지 않음

### 결정 사항
- **로딩 화면에 "프로필을 보내는 중" 문구**: "전송", "매칭 준비 중" 등 여러 후보 중 선택. "서버"나 "API"는 금지어이지만 "보내는 중"은 일반 구어라 AI 은폐 원칙과 충돌 없음. 매칭 단계 문구("파트너를 찾는 중")와 라임이 맞아 같은 톤
- **인사만 문장, 나머지 대화는 단어 유지**: 규칙을 본 직후에 규칙이 즉시 작동하기 시작하는 내러티브 — 플레이어 입장에서 "아, 세계 만들기가 진짜 시작됐구나"를 인지하는 트리거. 첫 인사가 단어였던 이전 버전은 규칙 학습 타이밍과 실제 적용 타이밍이 겹쳐 구분이 흐릿했음
- **`FALLBACK_GREETING` 폴백 문구**: 단어 2개짜리에서 문장 한 줄로 갈아치움. 실패 시에도 파트너가 "사람처럼" 말을 거는 톤 유지
- **"계속할까요?"는 물음표 유지, "다 만들었어요"는 마침표 없이 짧게**: 시스템 알림이지만 본문은 구어로 두어 차가움 완화. 이 공간이 여전히 두 사람 사이의 순간이라는 느낌
- **시스템 알림 스타일에 상하 경계선**: 단순 흐린 텍스트보다 "세션이 끊어졌다"는 시각적 구분이 강함. 대화 블록과 버튼 영역 사이의 경계로 기능
- **로딩 화면은 1.6초**: 너무 짧으면 무의미, 너무 길면 지루. 매칭 바(2.5초)보다 짧게 해 "프로필 검증 → 규칙 안내 → 매칭"의 호흡 계단이 점점 길어지도록 설계

### 미해결
- 첫 인사 문장이 규칙 화면의 "단어로만 소통" 규칙과 모순되어 보일 가능성. 프롬프트에 "지금은 시작 전"이라 명시했지만 모델이 혼동해 단어 배열로 회귀하면 파싱은 실패할 수 있음 → `FALLBACK_GREETING`로 자연 복구
- 세션 종료 시스템 알림의 한국어 letter-spacing이 실기기에서 너무 넓거나 좁아 보일 가능성. 실플레이 관찰 후 조정
