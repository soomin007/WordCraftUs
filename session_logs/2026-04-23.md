# 2026-04-23

## 요약

플레이테스트 피드백을 반영해 게임 언어를 "단어 전용"으로 재정립. 규칙 안내 화면, 단어 기반 파트너 응답, 세션마다 쌓이는 ASCII 아트 배경을 도입. 기본 모델을 flash-lite로 전환하고, 재시작 시 화면 잔여물이 남던 버그도 함께 수정.

## 주요 변경 (`index.html`)

### 1) 재시작 버그 수정 (`new-world-btn`)
- `chatPanel`/`worldLayersEl`/`session-end-choice`/`ascii-bg-container` 전부 초기화
- 세션 종료 후 숨겨뒀던 `input-area` / `input-hint` 복원, `wordInput.disabled = false`
- 이전엔 state만 리셋해서 이전 세계의 말풍선·WORLD 레이어·세션 종료 버튼이 새 플레이에 끌려 들어감

### 2) 규칙 안내 화면 (`rules-screen`)
- 흐름: `intro-screen` → `rules-screen` → `matching-screen`
- 공지 톤 박스(`NOTICE` 라벨 + 본문 + "준비되셨나요?" 질문 + `준비됐어요` 버튼)
- 본문은 `white-space: pre-wrap`으로 들여쓰기/줄바꿈 유지. 내용은 "단어로만 소통", "장소→사람→사건→유물", 각 단계 동작 설명
- `new-world-btn`에서는 규칙을 이미 봤으므로 `rules-screen`을 건너뛰고 `startMatching()`으로 직행 (의도적 선택)

### 3) 기본 모델 교체 (`MODELS`)
- `['gemini-2.5-flash-lite', 'gemini-2.5-flash']` — lite가 기본, flash가 폴백
- 이유: 사용자 요청. lite가 무료 티어에서 훨씬 덜 붐비고 세계 생성 수준에 충분. 품질 저하가 체감되면 flash가 받아줌

### 4) 파트너 응답을 단어로 (`partner_words` / `greeting_words`)
- **Gemini JSON 스키마 전면 변경**:
  - `partner_message` 삭제 → `partner_words: [3~5단어]`
  - `greeting` 삭제 → `greeting_words: [2~4단어]`
  - `ascii_art: "30자×15행 문자열"` 추가
  - `narrative` / `mood` 유지
- **프롬프트 전체 한국어화** (`buildPrompt`, `buildGreetingPrompt`)
  - "단어로만 소통" 규칙 명시, 좋은 예/나쁜 예 각각 두 세트
  - `priorAscii` 블록 주입: 이전 ASCII를 프롬프트에 넣어 "분위기를 이어서 발전시키라"고 지시
  - `PERSONAS[name].tone`과 `playerProfile`은 그대로 주입 유지
- **렌더링 변경**:
  - `addPartnerMsg`/`addMyMsg` → `addPartnerWords`/`addMyWords`. 둘 다 `formatWordList(words)` = `["w1", "w2"]` 형태로 출력
  - 파트너 말풍선도 플레이어 말풍선과 동일 스타일(테두리·JetBrains Mono)로 통일, 정렬만 좌/우로 대칭
  - narrative는 채팅 패널에서 제거 → WORLD 패널(`renderWorld`)에만 노출
  - 세션 종료 말풍선도 단어 배열(`['계속', '할까요']`, `['다', '만들었네요']`)
- **`NEXT_PROMPTS` 상수 삭제**: 2~4레이어 진입 시 파트너가 먼저 문장으로 유도하던 라인 제거. 다음 레이어 진입 후 플레이어 입력이 세션 시작점
- `tryJsonOnce` 검증 키도 `partner_words`로 교체

### 5) ASCII 아트 배경 누적 (`ascii-bg-container`, `addAsciiLayer`)
- `<div id="ascii-bg-container">`를 `<body>` 첫 자식, `position: fixed; inset: 0; z-index: 0`
- `.screen`에 `z-index: 1` 부여, `.chat-panel { background: rgba(13,13,13,0.85) }`, `.world-panel { background: rgba(13,13,13,0.60) }`, 두 패널 `position: relative; z-index: 1`
- 세션마다 새 `.ascii-layer` div를 누적. `color = mood.primary_color` 고정(레이어 생성 시 mood), `font-size`는 `asciiFontSizePx()`로 화면 꽉 차게 자동 계산 (30열 × 0.6 + 15행 비교)
- opacity 규칙: 최신 1.0, 바로 이전 0.35, 그 이전 0.2, 더 이전들 0.1. 새 레이어 추가될 때마다 `updateAsciiOpacities()`가 전체 재계산
- `transition: opacity 2s ease, color 2s ease`로 페이드인·색전환 부드럽게
- `window.resize`에서 `updateAsciiLayoutAll()`로 폰트 크기 재계산
- `new-world-btn`이 컨테이너를 비워 잔여 레이어 제거

## 결정 사항

- **단어로만 소통을 게임 언어로 격상**: 이전 버전은 파트너가 짧은 문장("오, 좋은데요?")을 뱉어 플레이어의 단어 입력과 비대칭했음. 이제 양측 모두 단어 배열로 말풍선 형태가 같아 시각적으로 '같은 규칙을 따르는 두 사람'처럼 보임
- **narrative는 채팅에서 제거, WORLD 패널 전용**: 채팅에 긴 묘사가 섞이면 단어 리듬이 깨짐. 묘사는 세계의 '결과물'로서 WORLD에만. 채팅은 인터랙션, WORLD는 세계. 역할 분리가 명확해짐
- **기본을 lite로 바꾸면서 프롬프트를 한국어로 재작성**: flash보다 instruction following이 약한 lite에 긴 영어 프롬프트를 주면 지시를 흘림. 한국어로 짧고 명시적으로 다시 쓰고, good/bad 예시를 모든 제약에 붙여 few-shot 유도
- **ASCII 아트를 "누적" 방식으로**: 교체가 아니라 겹치기. 초반엔 드문드문, 4세션 후에는 네 층이 겹쳐 빽빽해지는 시각적 성장 = 세계가 레이어로 쌓인다는 개념의 직관적 표현. 최신이 가장 진하고 이전 것이 그림자로 남는 opacity 차등은 플레이어가 '어디까지 만들었는지'를 스스로 느끼게 함
- **ASCII 레이어 color는 생성 당시 mood 고정**: 매번 최신 mood로 덮으면 네 층이 다 같은 색이 되어 누적 효과가 안 보임. 각 레이어가 그 시점 분위기를 간직하고, 전체는 겹쳐진 색 팔레트가 됨
- **패널 반투명: 채팅 0.85, WORLD 0.60**: 채팅은 글자 가독성이 최우선이라 더 불투명. WORLD는 세계 묘사라 배경이 더 비쳐 세계와 묘사가 한 화면에서 녹아 보이도록. 스펙 그대로
- **`new-world-btn`에서 `rules-screen` 스킵**: 규칙은 처음 오는 사람을 위한 onboarding. 새 세계 만들기를 누를 때는 이미 규칙을 아는 사용자. 재노출은 귀찮음만 늘림
- **폰트 크기 자동 계산에서 width/height 중 작은 값 채택**: 가로 기준 계산값이 세로 넘치면 잘리므로 안전한 쪽을 택함. 모바일 세로 화면에서도 30×15가 화면을 벗어나지 않음

## 미해결 / 다음 할 일

- **ASCII 30×15 스키마를 lite가 얼마나 지킬지 미검증**: 프롬프트에 "정확히 30자 × 15행"을 명시하고 이전 ASCII를 예시로 제공했지만, 행 길이가 들쭉날쭉할 가능성. 실플레이에서 깨지면 클라이언트 측 normalize(패딩/크롭) 추가 필요
- **ASCII에 줄바꿈 특수문자가 정확히 `\n`으로 올지**: JSON string에서 `\n`은 OK지만 모델이 `\\n`(백슬래시+n)로 쓸 수도 있음. 실측 후 재파싱 로직 고려
- **단어 기반 대화가 "대화처럼 느껴지는지" 실플레이 감각 체크**: 단어 배열만 오고 가면 오히려 기계적으로 읽힐 위험. 말풍선 타이핑 효과가 이를 얼마나 완화하는지, 타자 속도가 단어 길이에 비해 빠르게 느껴지는지 관찰
- **ASCII 전환 중 성능**: 4세션 누적 + 각 레이어에 opacity 2s 트랜지션 동시 작동. 모바일 저성능 기기에서 프레임 드롭 확인 필요
- **반전/공유 화면에 ASCII 배경이 남는 효과**: 의도된 연속성이지만, 반전 메시지("저는 사람이 아니에요")와 배경 ASCII가 시각적으로 경쟁할 수 있음. 필요하면 twist-screen 진입 시 컨테이너 opacity 감소 옵션 검토
