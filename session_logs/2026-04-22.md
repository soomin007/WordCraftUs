## 세션 2 — 자동 재시도 + 폴백 모델

### 맥락
라이브 플레이 중 Gemini `503 UNAVAILABLE` ("experiencing high demand") 발생으로 "다시 시도" 버튼을 반복해서 눌러야 하는 문제. 구조적으로 해결 + 파트너가 사람이라는 환상을 유지해야 함.

### 주요 변경 (`index.html`)
- 상수 추가: `MODELS = [flash, flash-lite]`, `RETRY_DELAYS = [1s, 2s, 4s]`, `RETRYABLE_STATUS = [408, 429, 500, 502, 503, 504]`, `STALL_MESSAGES`
- `callGemini` 재작성: 각 모델마다 최대 4회(1회 + 3회 재시도) 지수 백오프, 실패 시 폴백 모델로 전환. 영구 오류(4xx)는 즉시 중단. `onStall` 콜백으로 UI 문구만 갱신.
- `tryGeminiOnce` 보조 함수 추가 — 한 번의 호출 결과를 `{ok, retryable}`로 정규화.
- `showTypingIndicator` + `updateTypingIndicator` 분리. 재시도마다 "생각 중 → 조금만 기다려주세요 → 연결이 살짝 느리네요" 순으로 전환.
- 최종 실패 메시지를 "응답하지 않아요 → 다시 시도해볼게요"에서 "연결이 잠시 끊긴 것 같아요 → 다시 연결해볼까요?"로 변경.
- `validateApiKey`도 두 모델 순회. 기본 모델이 503이어도 키 유효성 확인 가능.

### 결정 사항
- **AI 정체 은폐 우선**: 재시도·폴백·HTTP 상태 등 기술 용어는 UI에 절대 노출하지 않음. "연결이 느리다"는 표현을 택한 이유: 실제 사람과의 온라인 플레이에서 자연스럽게 나올 법한 멘트라 몰입을 오히려 강화함.
- **폴백 모델로 `gemini-2.5-flash-lite` 선택**: 무료 티어에서 가장 덜 붐비는 플래그십 하위 모델. 품질 손실은 있지만 플레이 중단보다 나음.
- **재시도 횟수 4회·백오프 1-2-4s**: 일시적 폭주는 보통 수 초 내 해소됨. 너무 길면 사용자 인내심 초과, 너무 짧으면 스파이크를 못 피함.
- **`CLAUDE.md`에 두 가지 루틴 규칙 추가**: ① 수정→커밋→푸시→세션 로그 업데이트를 한 묶음으로 ② AI 정체 은폐 (UI에 기술 용어 금지). 사용자가 매번 상기시키지 않아도 지켜지도록.

### 미해결
- `gemini-2.5-flash-lite`가 JSON 응답 스키마를 얼마나 안정적으로 지키는지 실제 플레이에서 확인 필요 (플래시보다 추론력이 떨어져 구조 어길 가능성)
- 재시도 중 사용자가 창을 닫거나 네트워크를 끊으면 핸들이 남는지 점검 (현재는 abort controller 없음)

## 세션 3 — "사람답게" 튜닝 (프롬프트 + 타자 연출 + 재시도 문구)

### 맥락
플레이 테스트 결과 두 가지 부자연스러움이 드러남:
1. 파트너가 뱉는 문장이 너무 문학적·장문이라 "이건 사람이 채팅으로 쓴 게 아닌데" 티가 남
2. 재시도 문구가 시스템 안내 느낌 ("연결이 살짝 느리네요")이라 컨셉에 맞지 않음

### 주요 변경 (`index.html`)

**프롬프트 재설계 (`buildPrompt`)**:
- 역할을 "creative writing partner"에서 "친구랑 채팅으로 세계 만드는 보통 사람"으로 재정의
- "NO literary/poetic language", "NO ornate metaphors" 등 명시적 금지 조항
- 나쁜 예시(현재 실제로 생성됐던 문장)와 좋은 예시를 프롬프트에 내장 — few-shot으로 톤 고정
- `narrative`: 1-2문장, ~60자 제한
- `partner_message`: ~25자 이하 한 문장. 구체 예시 제공 ("오, 좋은데요?" 등)

**타자 연출 (`typewriter`)**:
- 기본 속도 15ms → 40ms
- 글자마다 ±15ms 랜덤 편차 (최소 10ms 하한)
- `.`, `,`, `!`, `?`, `…`, `。`, `，` 등 구두점 뒤에 250ms 추가 멈춤
- → 실제 사람이 채팅으로 타이핑하다가 생각하는 듯한 리듬

**재시도/실패 문구 (`STALL_MESSAGES`, 최종 실패 메시지)**:
- "연결이 살짝 느리네요" (시스템 안내 느낌) → 제거
- "${name}가 잠깐 멈춘 것 같아요, 기다려볼게요" 등 **파트너가 잠시 멈춘 것을 플레이어가 관찰하는 시점**의 문구로 교체
- 최종 실패: "연결이 끊긴 것 같아요" → "한참 조용하네요. 다시 한 번 말 걸어볼까요?"
- 금지 어휘 주석으로 못박음: "연결", "응답", "서버", "재시도", "모델"

**`addPartnerMsg` 개선**:
- 이전엔 `typewriter`를 fire-and-forget 하고 `await sleep(length × SPEED)`로 대충 기다림 → 속도·편차 변경에 취약
- 이제 Promise를 반환하여 `await addPartnerMsg(...)`로 정확히 대기 후 `sleep(1200)`만 추가

### 결정 사항
- **프롬프트에 실제 나쁜 예시를 박음**: 추상적인 "간결하게" 지시만으로는 Gemini가 계속 문학체로 회귀. 과거 실제 생성문을 bad example로 주입하니 톤이 즉시 바뀌는 패턴 활용.
- **타자 속도 40ms는 "천천히 타이핑하는 사람" 기준**: 숙련 타이피스트(~200 WPM)는 12ms/글자까지 가능하지만, 채팅에서 생각하며 치는 일반 사람은 30-60ms. 불규칙성이 더 중요.
- **구두점 멈춤은 쉼표/마침표 공통 250ms**: 쉼표는 더 짧게 할 수도 있지만, 단순화 우선. 체감상 충분히 '사람같음'.
- **재시도 문구를 "파트너 관찰" 시점으로**: 사용자 예시 "기다려볼게요"를 살림. 플레이어가 파트너의 침묵을 관찰하는 뉘앙스 → 시스템 느낌 완전히 제거.

### 미해결
- 프롬프트 톤 변경 후 `gemini-2.5-flash-lite`가 여전히 격식체로 돌아가는지 관찰 필요 (lite는 instruction following이 상대적으로 약함)
- 한 번에 한 메시지만 오도록 했지만, 사람이면 짧은 메시지를 2-3개 연속으로 보내는 경우도 있음. 나중에 "chunk split" 연출 검토 가능
