## 결정 사항

- **API 키 관리 방식**: 원본 스펙은 "플레이어가 시작 화면에서 키 입력 → sessionStorage"였으나 사용자 요청으로 `config.js` + GitHub Actions 시크릿 주입 방식으로 변경. 이유: 플레이어가 키를 준비하지 않아도 바로 플레이 가능하게 하기 위함. 레포에는 키가 노출되지 않고 라이브 사이트는 정상 동작.
- **단일 파일 원칙 유지**: 스펙대로 `index.html` 하나에 모든 것을 담음. 외부 JS/CSS 파일 분리하지 않음. 배포 단순성 우선.
- **세션 화면 레이아웃**: 채팅 + 세계 패널을 2단 그리드로. 모바일은 자동으로 2행 세로 배치.
- **반전 화면 연출**: 문장별로 1.4초 간격 opacity fade-in. 스펙의 "천천히 페이드인" 톤에 맞춤.
- **상위 git 레포 회피**: `C:/Users/soomi`가 의도치 않은 git 레포로 잡혀 있어서 `WordCraftUs` 안에 독립 `git init`. 이쪽이 `soomin007/WordCraftUs` 리모트에 연결됨.
